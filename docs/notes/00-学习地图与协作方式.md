# 学习地图与协作方式

> 本项目的目标不是只做出桌宠，而是借桌宠学会一个小型桌面应用从设计、实现到调试的完整过程。

---

## 一、项目学习主线

这个项目最重要的主线是：

```text
数据从哪里来 → 经过谁处理 → 怎么变成用户看到的界面
```

只要这条线清楚，后面的设置、提醒、托盘、换肤、状态机和 AI 对话，都是在这条主线上继续加功能。

---

## 二、推荐学习阶段

### 阶段 1：让页面动起来

重点理解：
- HTML 结构
- CSS 形状、颜色和动画
- JS 事件监听
- `requestAnimationFrame`

对应文件：
- `renderer/index.html`
- `renderer/css/pet.css`
- `renderer/js/pet.js`

### 阶段 2：让 Electron 跑起来

重点理解：
- `main.js`
- `BrowserWindow`
- 透明窗口、置顶窗口
- `preload.js`
- 主进程和渲染进程的区别

### 阶段 3：让数据进来

重点理解：
- Node.js `fs` 模块
- `fs.watch()`
- JSON 文件结构
- Claude Code 数据来源
- IPC 数据传递

对应文件：
- `cc-monitor.js`
- `preload.js`
- `renderer/js/detail.js`

### 阶段 4：让架构变清楚

重点理解：
- `app.js` 事件总线
- 模块拆分
- 单向数据流
- “谁负责什么，不负责什么”

目标方向：

```text
cc-monitor -> app.js -> bubble/detail/settings 等 UI 模块
```

## 三、每次改动的学习记录格式

遇到问题或新增功能时，优先按这个格式记录：

```text
我看到的现象是什么？
真正原因是什么？
我学到了哪个通用知识点？
```

示例：

```text
现象：启动后直接显示详情面板
原因：CSS display:flex 覆盖了 hidden
知识点：HTML 属性和 CSS 共同决定最终显示效果
```

如果这个知识点以后还会用到，就写进 `docs/notes/`。

---

## 四、推荐开发节奏

不要一开始追求完美架构。更适合初学者的节奏是：

```text
先让功能跑起来
再解释为什么能跑
再整理成学习笔记
最后再重构结构
```

这样能先看到效果，再理解原理，最后再把代码整理得更清楚。

---

## 五、协作方式约定

后续修改项目时，应尽量按这个流程推进：

1. 先说明这次要解决什么问题
2. 指出涉及哪些文件
3. 解释会用到的知识点
4. 改一小步
5. 启动或检查验证
6. 总结是否需要新增学习笔记

这个项目是学习型项目。代码能跑只是第一步，真正重要的是理解每个改动为什么这样写。
