# 基础概念（知识点笔记）

> 学习桌宠开发过程中遇到的核心概念，用大白话记录。

---

## Node.js

**不是什么**：不是编辑器、不是框架、不是编程语言。

**是什么**：一个让 JavaScript 能在浏览器**之外**运行的环境。

**为什么桌宠项目需要它**：
- 我们要用 Electron 做桌面应用 → Electron 由 Node.js 启动
- 我们要装第三方库 → npm（Node.js 自带的包管理器）负责下载安装
- Claude Code 本身也是 Node.js 程序

**一句话记**：
> 浏览器让 JS 能在网页里跑，Node.js 让 JS 能在电脑上跑。

---

## JavaScript（JS）

**不是什么**：和 Java 完全无关。（名字是 1995 年为了蹭 Java 热度改的，实际上两个语言天差地别。）

**是什么**：让网页"活起来"的语言。

**三件套比喻**：

| 技术 | 比喻 |
|------|------|
| HTML | 人的骨架 |
| CSS | 衣服和化妆 |
| JavaScript | 肌肉和神经——让它动起来 |

**在本项目中的用途**：控制宠物眨眼、走路、冒气泡、读取 Claude Code 数据。

---

## Electron

**是什么**：一个桌面应用开发框架。

**做什么**：把你的网页代码（HTML/CSS/JS）变成真正的桌面软件。

**能力**：
- 创建桌面窗口（透明、无边框、置顶）
- 系统托盘（右键菜单）
- 读写本地文件
- 调用 Node.js 所有能力

**常见 Electron 软件**：VS Code、Discord、Slack、Figma、Obsidian。

**注意事项**：Electron 在 Windows 上存在一个已知 bug——`require("electron")` 有时会返回路径字符串而不是真正的 API。这是 GitHub Issue #49034，截至目前（2026年7月）仍在修复中。

---

## package.json

**是什么**：Node.js 项目的"身份卡 + 购物清单"。

**作用**：
- 告诉电脑这个项目叫什么、怎么启动
- 列出项目需要哪些依赖库
- 别人拿到项目后，执行 `npm install` 就能装好所有依赖

**关键字段**：

```json
{
  "main": "main.js",        // 入口文件，启动时先跑这个
  "scripts": {
    "start": "electron ."   // 快捷命令，npm start = 执行 electron .
  },
  "devDependencies": {
    "electron": "^30.0.0"   // 开发依赖（运行时不一定要）
  },
  "private": true           // 防止误发布到 npm 官网
}
```

---

## 依赖管理（npm install）

**执行 `npm install` 时发生了什么**：

```
你写 package.json → 声明"我需要 electron"
        ↓
执行 npm install
        ↓
npm 去 registry 下载 electron → 放到 node_modules/
        ↓
electron 的 postinstall 脚本再去下载二进制文件（~130MB）
        ↓
node_modules/electron/dist/ 里出现 electron.exe
```

**注意**：`node_modules/` 不要手动改，也不要上传到 Git（已在 `.gitignore` 中排除）。

## 依赖的两种类型

```json
{
  "dependencies": {     // 用户运行软件时需要
    "some-lib": "^1.0"
  },
  "devDependencies": {  // 开发时才需要
    "electron": "^30.0" // Electron 按惯例放 devDependencies
  }
}
```

## require() 的工作原理

```js
const { app, BrowserWindow } = require("electron");
```

`require()` 是 Node.js **引入模块** 的方式。它的查找顺序：
1. 看是不是 Node.js **内置模块**（如 `fs`、`path`）
2. 看是不是相对路径（如 `./xxx`）
3. 去 `node_modules/` 里找

### 关于 electron 包的特殊说明

**`electron` 这个 npm 包的 index.js 只导出一个东西——Electron 可执行文件的路径字符串。**

```js
// node_modules/electron/index.js（简化）
module.exports = getElectronPath();  // 返回 "path/to/electron.exe"
```

这意味着：
- **在普通 Node.js 里** `require("electron")` → 得到路径字符串 ✅ **这是正常行为**
- **在 Electron 运行时里** `require("electron")` → 应该被拦截，返回真实 API

**关于我们遇到的 Windows 兼容问题**：
- Electron v31+ 在某些 Windows 系统上，运行时拦截 `require("electron")` 的机制失效了
- 表现为：即使在 Electron 里跑，`require("electron")` 仍然返回路径字符串
- 这是 Electron 官方的 Windows bug（GitHub Issue #49034）
- 开发时的临时方案：在 `main.js` 里用 `typeof electron === "string"` 检测，自动用 `child_process.spawn` 转交给 Electron
- 同时用 `process.versions.electron` 做防递归保护

## CSS 伪元素与皮肤架构

```html
<!-- 错误做法：眼睛写在 HTML 里，换皮肤要改结构 -->
<div id="pet">
  <div class="eye left"></div>
  <div class="eye right"></div>
</div>

<!-- 正确做法：HTML 只有容器，外观由 CSS 决定 -->
<div id="pet"></div>
```

```css
/* 用伪元素画眼睛，皮肤 = 换一套 CSS */
#pet::before { }   /* 左眼 */
#pet::after { }    /* 右眼 */
```

**好处**：HTML 和 JS 不用动，换皮肤就是换 CSS。V2 更彻底——用图片序列帧包，连 CSS 都不用改。

## 什么是"事件驱动"

**不是**：
```js
// ❌ pet.js 直接调 bubble.js — 互相引用，改一个要改一串
pet.js:   bubble.show("上下文满了")
detail.js: bubble.show("上下文满了")
```

**而是**：
```js
// ✅ 各模块只发事件，app.js 统一转发
pet.js:   app.emit("alert", {type: "context-full"})
detail.js: app.emit("alert", {type: "context-full"})
app.js:   app.on("alert", data => bubble.show(data))
```

**好处**：加功能只需要在 `app.js` 加一条规则，其他文件不用动。
