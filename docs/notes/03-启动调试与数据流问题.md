# 启动调试与数据流问题

> 记录桌宠启动检查中遇到的几个关键知识点。

---

## 1. `hidden` 为什么可能失效

HTML 里写：

```html
<div id="detail-panel" hidden>
```

表示元素默认隐藏。但如果 CSS 又写了：

```css
#detail-panel {
    display: flex;
}
```

这个 `display: flex` 会把隐藏效果覆盖掉，导致详情面板启动时直接显示。

修复方式：

```css
#detail-panel[hidden] {
    display: none;
}
```

**记住**：最终显示效果由 HTML 属性和 CSS 共同决定。遇到“明明 hidden 了还显示”，先查 CSS 的 `display`。

---

## 2. 文件名匹配要看真实格式

真实备份文件名类似：

```text
.claude.json.backup.1782922105706
```

如果代码写：

```js
filename.endsWith(".json.backup")
```

会匹配失败，因为文件名后面还有时间戳。

更适合当前格式的写法：

```js
filename.includes(".json.backup")
```

**记住**：先观察真实文件名，再决定用 `endsWith`、`startsWith` 还是 `includes`。

---

## 3. 读 JSON 前先看结构

备份文件里的项目数据不在顶层，而在：

```js
data.projects["E:/kaifa/animal"]
```

不是：

```js
data["E:/kaifa/animal"]
```

调试 JSON 时可以先打印：

```js
console.log(Object.keys(data));
console.log(Object.keys(data.projects));
```

**记住**：不要凭想象读 JSON。先看结构，再写访问路径。

---

## 4. IPC 监听可能错过第一次消息

主进程可能很早就发送数据，但页面脚本还没完成监听注册。结果就是：数据发过了，页面没听到。

稳妥做法是：

```js
window.ccAPI.onUpdate(handleUpdate);
window.ccAPI.refresh();
```

先注册监听，再主动请求一次最新数据。

**记住**：事件推送适合实时更新，主动刷新适合补齐首次状态。

---

## 5. 计算用原始数据，展示再格式化

错误思路：

```js
const inputTokens = total.toLocaleString();
const percent = inputTokens / 200000;
```

`toLocaleString()` 会把 `367862` 变成 `"367,862"`，这个字符串再参与计算可能得到 `NaN`。

正确思路：

```js
const rawInputTokens = total || 0;
const inputTokens = rawInputTokens.toLocaleString();
const percent = rawInputTokens / 200000;
```

**记住**：计算用数字，展示用字符串。
