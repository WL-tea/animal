# Windows 系统托盘图标设计

> 日期：2026-07-15  
> 关联 Issue：#28  
> 状态：MVP 草稿方案，允许后续整体替换

## 目标

为 Windows 系统托盘提供一套清晰、稳定、可替换的专用图标资源。图标延续当前桌宠的陶土橙圆形身体和深色双眼，并增加极简短耳朵，使其在小尺寸下仍具有角色辨识度。

本设计只负责图标资产及其导出规范，不在本次工作中创建 Electron `Tray`、接入托盘菜单或修改窗口生命周期。这些消费逻辑由后续 Issue #29 负责。

## 视觉方案

采用实心圆脸徽标：

- 主体为陶土橙圆脸，使用当前桌宠强调色 `#D4784C`；
- 顶部增加两只短耳朵，轮廓保持圆润，避免形成复杂尖角；
- 双眼使用当前近黑色 `#1E1E20`；
- 外缘增加克制的浅色描边，兼顾 Windows 浅色和深色任务栏；
- 不使用文字、渐变、投影、状态圆点或 Agent 标记；
- 主体居中，四周保留约 12% 安全边距，避免托盘裁切和视觉拥挤。

这是 MVP 草稿，不作为最终品牌或皮肤规范。后续可以替换图像内容，但应保留稳定的文件职责和导出尺寸，以免托盘代码随视觉升级而修改。

## 资产结构

最终资源放在 `assets/tray/`：

```text
assets/tray/
├── tray-icon-source.png
├── tray-icon-16.png
├── tray-icon-20.png
├── tray-icon-24.png
├── tray-icon-32.png
├── tray-icon-40.png
├── tray-icon-48.png
├── tray-icon-64.png
├── tray-icon-128.png
└── tray-icon-256.png
```

`tray-icon-source.png` 是透明背景的高分辨率母图，其余文件是从同一母图生成的透明 PNG。消费代码不依赖生成过程，只依赖约定的稳定路径。

## 生成与导出流程

1. 生成正方形母图，使用完全平坦的纯色背景，便于移除背景。
2. 本地移除背景并验证透明通道、透明四角和主体边缘。
3. 从透明母图按 16、20、24、32、40、48、64、128、256 像素导出。
4. 使用高质量缩放；逐张检查主体占比、眼睛是否可辨、耳朵是否粘连以及是否存在色边。
5. 保留无状态的单一基础图标。正常、告警、暂停等变体不属于本次 MVP 范围。

## 验证标准

自动检查：

- 每个 PNG 的宽高符合文件名；
- 每个 PNG 均包含 alpha 通道；
- 四角透明，主体像素覆盖率合理；
- 所有派生尺寸来源一致，文件名稳定；
- `git diff --check` 通过。

视觉检查：

- 16px 下仍可看出圆脸、双眼和耳朵；
- 100%、125%、150%、175%、200% Windows 缩放对应的常见显示尺寸下没有明显糊边；
- 在浅色和深色背景预览中都有足够轮廓对比；
- 透明边缘没有明显的纯色背景残留。

真实 Windows 托盘截图需要在 #29 创建 `Tray` 并接入资源后完成。本次先验证静态资源本身，并明确尚未覆盖的系统级验收。

## 错误与替换边界

- 本次不增加运行时资源加载，因此不会引入启动失败路径；
- #29 接入时必须处理资源缺失，不能让图标文件错误导致应用无法启动；
- 后续视觉替换应重新生成全部尺寸，不能只替换母图而留下旧派生文件；
- 图标资源不包含项目路径、状态详情或其他本机隐私信息。

## 非目标

- 动画托盘图标；
- 告警、暂停等状态变体；
- 安装包图标、可执行文件图标或普通窗口图标；
