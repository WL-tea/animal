# 桌宠项目设计文档

> 2026-07-01 · 设计阶段

---

## 一、产品定位

一个**功能型桌面宠物**，悬浮于桌面上层，能自主走动、与用户互动，实时展示 Claude Code 运行状态，并支持定时提醒。核心原则：**宠物是主角，信息通过宠物的"表达"来传递**，而非在宠物身上贴文字。

目标用户：使用 Claude Code 进行开发的程序员，可能同时管理多个项目。

---

## 二、技术栈

| 项目 | 选项 | 理由 |
|------|------|------|
| 框架 | Electron | 透明窗口 + `alwaysOnTop` + Node.js 原生文件监听 |
| 渲染 | HTML + CSS + JS | 轻量、学习曲线平、AI 辅助质量高 |
| 架构 | 轻量单页 | 先让宠物跑起来，V2 后再模块化 |
| 打包 | electron-builder | 一键生成 exe |

**Electron 主进程职责**：创建透明无边框窗口、管理系统托盘、启动 CC 文件监听、读写用户设置。渲染进程负责所有 UI 和动画。

## 三、交互模型（三层架构）

```
┌──────────────────────────────────────────┐
│  层1：宠物本体                            │
│  · 纯视觉角色，无文字叠加                 │
│  · 走动、眨眼、待机动画                   │
│  · 可拖拽移动                             │
│                                          │
├──────────────────────────────────────────┤
│  层2：对话气泡                            │
│  · 临时弹出，3 秒自动消失                 │
│  · 简短消息（CC 告警、问候）              │
│  · 点击气泡 → 打开详情窗口                 │
│                                          │
├──────────────────────────────────────────┤
│  层3：详情窗口 + 设置窗口                 │
│  · 用户主动打开                           │
│  · 完整 CC 数据仪表盘                     │
│  · 设置面板独立入口                       │
└──────────────────────────────────────────┘
```

**打开详情窗口的方式**（优先级从高到低）：

1. 双击宠物
2. 点击对话气泡
3. 右键宠物 → 菜单
4. 系统托盘图标

## 四、Claude Code 监控

### 数据来源

| 数据 | 源文件 | 实时性 |
|------|--------|:---:|
| Token 消耗 | `~/.claude/projects/<项目>/<会话>.jsonl` | 实时 |
| 聚合统计 | `~/.claude/backups/.claude.json.backup.*` | 定期 |
| 会话状态 | `~/.claude/sessions/<pid>.json` | 实时 |
| Agent 活动 | `~/.claude/projects/<项目>/<会话>/subagents/` | 实时 |
| 任务列表 | `~/.claude/tasks/<会话>/` | 实时 |

监控方式：`fs.watch()` 监听 `.claude` 目录，JSONL 逐行解析。

### 自动告警事件

| 事件 | 触发条件 | 气泡文案 |
|------|---------|---------|
| 上下文快满 | 任意项目上下文 > 用户设定阈值（默认 80%） | 「上下文快满了，该 compact 了~」 |
| 上下文危急 | 上下文 ≥ 95% | 「要爆了！快去处理！」 |
| Agent 完成 | 所有子 Agent 退出 | 「Agent 都跑完啦~」 |
| 会话空闲 | 会话 busy → idle | 「休息中...需要我帮忙吗？」 |
| 花费超标 | 当日消耗 > 用户限额 | 「今天花了 $X 了哦」 |
| 新会话 | 监控项目新会话启动 | 「开工了！项目 X 有动静~」 |
| 项目卡住 | 某项目长时间无响应 | 待定 |
| 任务进度 | 任务状态变化 | 「又完成了一个任务！」 |

可在设置中开关每个事件、调整阈值。

## 五、宠物设计

### MVP 阶段
- 纯 CSS 绘制的几何角色（圆体 + 眨眼动画）
- 不依赖任何图片资源
- 状态：待机 → 走动

### 状态机（V2 完整版）

```
  待机 ←→ 走动
   ↓       ↓
  睡觉（用户长时间不互动）
  警觉（CC 告警时）
```

### 走动行为
- 随机方向、随机间隔
- 走动范围用户自定义（可视化虚线框拖拽，默认全屏）
- 碰到边界弹回
- 可拖拽到任意位置

### 置顶模式
- 始终置顶（默认）
- 仅通知时置顶（V2）

## 六、详情窗口

### 布局
侧栏项目列表 + 右侧主区域，适配 1~10+ 个项目。

### 配色系统

| 角色 | 色值 | 说明 |
|------|------|------|
| 页面背景 | `#FAF8F5` | 暖纸色 |
| 卡片/内容区 | `#FFFFFF` | 纯白 |
| 主文字 | `#1E1E20` | 近黑 |
| 辅助文字 | `#787880` | 中灰 |
| 标题栏 | `#2C2C30` | 深灰 |
| 强调色 | `#D4784C` | 陶土橙 |
| 在线状态 | `#5B9661` | 鼠尾草绿 |
| 告警状态 | `#D4A040` | 琥珀黄 |

### 字体
- 标题：sans-serif 18px / weight 600
- 数据：等宽字体（token 数字）
- 标签：12px / weight 400 / letter-spacing 0.5px

### 签名元素
右下角 8px 状态圆点：绿呼吸（在线）、灰静止（离线）、黄闪烁（告警）。

## 七、设置面板

### 入口
右键宠物 / 详情窗口右上角齿轮 / 系统托盘

### 配置项

**宠物行为**
- 活动范围（可视化拖拽虚线框）
- 走动频率（活跃 / 正常 / 慵懒）
- 置顶模式（始终 / 仅通知时）
- 宠物大小

**CC 监控**
- 添加/移除监控项目路径
- 刷新间隔
- 告警事件开关 + 阈值

**外观**（V2）
- 皮肤选择

## 八、MVP 项目文件结构

```
animal/
├── package.json
├── main.js                    # Electron 主进程
├── preload.js                 # IPC 安全桥接
├── renderer/
│   ├── index.html             # 入口
│   ├── css/
│   │   ├── pet.css
│   │   ├── bubble.css
│   │   ├── detail.css
│   │   └── settings.css
│   ├── js/
│   │   ├── app.js             # 主控 + 事件总线
│   │   ├── pet.js             # 宠物行为
│   │   ├── pet-states.js      # 状态机
│   │   ├── bubble.js          # 气泡逻辑
│   │   ├── detail.js          # 详情窗口
│   │   ├── settings.js        # 设置面板
│   │   └── cc-monitor.js      # CC 数据读取
│   └── assets/
│       └── icons/
├── data/
│   ├── settings.json          # 运行时生成
│   └── projects.json
└── README.md
```

### 模块通信铁律

1. **数据单向流动**：cc-monitor → app.js → 各 UI 模块
2. **事件驱动**：模块之间通过 `app.js` 事件总线通信，不直接互相调用
3. **每个 .js 文件职责单一**，加功能 = 在 `app.js` 加转发规则

## 十、技术学习路线

实现 MVP 需要的 JavaScript 知识点（按出现顺序）：

1. **变量、函数、事件监听** — `pet.js` 基础
2. **定时器 `setInterval` / `requestAnimationFrame`** — 宠物动画
3. **DOM 操作** — 气泡、详情窗口
4. **Node.js `fs` 模块** — CC 数据读取
5. **Electron IPC** — 主进程 ↔ 渲染进程通信
6. **JSON 解析与存储** — 用户设置持久化

每个知识点在实际需要时才学，边做边懂。
